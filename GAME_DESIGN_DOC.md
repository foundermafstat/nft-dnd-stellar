# NFT-DND: Процедурная многопользовательская RPG в реальном времени
## Технический и Геймдизайн Документ (GDD & TDD)

---

## Введение
**NFT-DND** — это процедурная многопользовательская RPG в реальном времени с браузерным клиентом, где ключевые элементы игрового процесса (лор, структура квестов, параметры NPC, лут) динамически генерируются ИИ-Мастером (AI Dungeon Master). Игра глубоко интегрирована с блокчейном **Stellar** для чеканки NFT-предметов, фиксации достижений (On-chain history) и обеспечения честности/приватности игровых механик за счет ZK Proofs (Protocol 25 / X-Ray).

---

## 1. Архитектура ИИ-Мастера

Центральный мозг игры, работающий как набор распределенных сервисов (AI Pipeline), который анализирует стейт мира и генерирует контент.

### 1.1. Компоненты ИИ-Мастера

| Компонент | Назначение | Техническая реализация (Рекомендация) |
| :--- | :--- | :--- |
| **LLM-ядро** | Генерация диалоговых веток, лора, описания квестов. | Прямая или прокси-интеграция с OpenAI API (GPT-4o) / Anthropic (Claude 3.5 Sonnet) с жестко заданным System Prompt. |
| **Narrative Director** | Управление контекстом, темпом и состоянием сюжета для каждой Party. Оценка "интересности" сессии. | Векторная база данных (Pinecone/PostgreSQL+pgvector) для RAG (Retrieval-Augmented Generation) и памяти ИИ. |
| **Adaptive Difficulty Engine** | Динамическая регулировка сложности (характеристики мобов, ловушки) в реальном времени. | Математическая/Heuristic Rule Engine (оценка DPS, Win/Loss ratio и среднего Gear Score группы). |
| **World State Tracker** | Хранит X/Y координаты, активные триггеры, состояние всех NPC. | In-memory БД (Redis) для real-time доступа (100k+ ops/sec) + асинхронный сброс в основной сторадж PostgreSQL. |
| **Player Behavior Analyzer** | Оценка стиля игры (агро, стелс, дипломатия, жадность). | Алгоритм скоринга действий и текстовых инпутов. Обновляет профиль (Player Vector) каждые 5 минут. |
| **Loot & Lore Ming. Engine** | Создание уникального лута на основе сюжета. | Сервис генерации JSON-метаданных, отправки в IPFS (Pinata) и вызова Soroban смарт-контракта. |
| **ZK-Validation Layer** | Валидация скрытых действий игроков. | Service-worker, использующий Circom/SnarkJS для генерации и проверки ZK-снарков. |

### 1.2. Логика поведения ИИ

*   **Реакция на решения:** ИИ принимает на вход стейт `[{PlayerAction}, {CurrentState}]` и возвращает четко типизированный JSON `[{NewStateModifiers}, {SystemMessage}, {NextOptions}]`.
*   **Сюжет в реальном времени:** У ИИ-Мастера есть "План А" (скелет квеста, сгенерированный заранее). При критических отклонениях (например, игроки убили сюжетного NPC) ИИ запускает "Фазу Перестроения", генерируя нового злодея, штраф (Bounty) или последствия, не нарушая общую канву.
*   **Ключевые входные данные:** Текущее HP/MP группы, пул предметов в инвентаре, последние 5 выборов в диалогах, время на локации (Time-To-Kill), публичные ключи Stellar (для проверки On-chain кармы).
*   **Предотвращение тупиков (Dead-ends):** Заложен "Костыль Судьбы" (Fate Fallback). Если группа застряла на локации >10 минут (нет прогресса по метрикам), Narrative Director спавнит странствующего торговца-подсказчика, запускает обвал пещеры (открывая новый проход) или провоцирует нападение патруля мобов, на теле которых будет карта.
*   **Баланс (Хаос vs Сюжет):** 70% генерации ограничено жесткими процедурными шаблонами (Procedural Generation Rules — структура комнат, базовые статы). 30% отдано на креативность LLM (описания, диалоги, "Флавор", уникальные перки предметов). Этим предотвращаются "галлюцинации" ИИ, ломающие баланс.

### 1.3. Генерация контента (Пайплайн)

**NPC (Генерация происходит в момент входа в новую зону):**
*   Тип (торговец, стражник, монстр, босс).
*   Характер (агрессивный, трусливый, алчный).
*   Цели (защищать, украсть, выжить).
*   Секреты/Слабости: "Боится огня", "Ищет пропавшую дочь" (используется для дипломатического прохождения).
*   Реплики: Многоуровневое древо, которое LLM генерирует на 3-4 уровня вглубь.

**События:**
*   *Рандомные:* (5%) Падение метеорита (изменение клеток ландшафта, спавн ресурса).
*   *Контекстные:* (80%) Засада разбойников, спровоцированная тем, что у игроков много золота в инвентаре.
*   *Глобальные:* (15%) Вторжение Мирового Босса (задействует весь онлайн сервера в пределах инстанса).

**Квесты (Формальная структура):**
Каждый квест генерируется как граф (Quest Graph). ИИ выстраивает Путь: Цель -> Препятствие -> Мини-босс -> Развилка (Выбор пути) -> Главный Босс -> Награда (с лором, привязанным к этому пути).

**Лор мира:**
ИИ может связывать ранее сгенерированных персонажей и предметы в цельную историю (используя RAG). Например, Меч, найденный 2 дня назад другим игроком в этой зоне, будет упомянут местным кузнецом в диалогах как "Украденная Реликвия".

## 2. Система игрового мира

Минималистичный 2D top-down мир (схематичный стиль или пиксель-арт на Canvas API/WebGL). Сервер авторитетно просчитывает мир.

### 2.1. Правила Procedural Generation
*   **Зоны и биомы:** Мир представляет собой 2D сетку (Hex Grid или Tilemap). Каждая зона имеет свой биом (Dark Forest, Crystal Caves).
*   **Параметры генерации:** При старте сервера генерируется граф комнат с помощью алгоритмов (Random Walk / Cellular Automata).
*   **Ключевые параметры карты:** Уровень Угрозы (1-10), Плотность Ресурсов (0.1-1.0), Магический Фон (Влияет на мана-реген).
*   **Механика масштабирования (Scale-up):** Если в биом заходит много людей, он динамически "расширяется" — ИИ достраивает новые подзоны "на лету" алгоритмом шума Перлина (Perlin Noise), после чего расставляет узлы интереса (POI).

### 2.2. Предопределённые шаблоны (Библиотеки)
Для предотвращения хаоса, ИИ использует кирпичики:
*   **Типы комнат:** `[Arena]`, `[Puzzle_Room]`, `[Treasure_Room]`, `[Corridor]`, `[Safe_Zone]`.
*   **Типы мобов:** `[Swarm]`, `[Tank]`, `[Caster]`, `[Boss]`. У каждого есть пресеты (HP, ATK, SPD, RNG), которые ИИ скалирует множителями.
*   **Структура подземелий:** 1 Вход -> 3 Комнаты с мобами -> Развилка -> Загадка/Ловушка -> Босс-рум.
*   **Событийные цепочки:** Шаблон `[Спасение_NPC] -> [Предательство] -> [Скрытая_Битва]` или `[Поиск_Ключа] -> [Загадка] -> [Сверх-награда]`.

### 2.3. Реальный-временной цикл (Tick Rate & Loop)
*   **Server Physics Tick:** 10-20 TPS (Ticks Per Second) для передвижения, коллизий и боевых расчетов.
*   **AI Director Tick (Heartbeat):** 1 раз в 5 секунд. ИИ сканирует состояния групп.
*   **World Updater Tick:** Раз в 60 секунд обновляются глобальные таймеры (респавн базовых ресурсов, погода). Синхронизация: Состояния рассылаются клиентам через WebSockets (Delta compression — отправка только изменений).

## 3. Многопользовательская система (Real-time)

### 3.1. WebSockets-службы
Используется Socket.io / uWebSockets.js (или Rust Tungstenite) для максимальной отзывчивости.
*   `channel_movement`: Позиции `[player_id, x, y, direction]`. Оптимизировано через бинарные буферы.
*   `channel_combat`: События `[actor_id, target_id, damage, skill_id, is_crit]`.
*   `channel_ai_narrative`: Потоковая передача (Streaming) сгенерированного лора или диалогов (посимвольное появление у клиентов).
*   `channel_voting`: Синхронизация таймеров голосования и текущих результатов.
*   `channel_spawns`: Генерация новых квестов/объектов на карте (отправка JSON-схемы объекта).

### 3.2. Lobby & Party-механики
*   **Группы (Party):** От 1 до 5 игроков. У каждой группы свой инстансный `Session_ID`.
*   **Роли (Roles):** Явных классов нет, распределение ролей негласное (кто носит щит — танк, кто посох — Caster/Хиллер). ИИ анализирует надетый Gear игроков в группе.
*   **Кооперативные задачи:** ИИ генерирует пазлы: "Игрок 1 должен стоять на нажимной плите (Требование STR > 20), пока Игроки 2 и 3 сдерживают волну `[Swarm]`, а Игрок 4 с INT > 25 читает древнюю руну для деактивации защиты Босса".

### 3.3. PvE-боёвка
*   **Модель урона:** `Final_Damage = (BaseATK * SkillMultiplier * TypeEffectiveness) - (Armor * PenetrationModifier)`.
*   **Алгоритм мобов:** Поведение на основе конечных автоматов (FSM) + Behavior Trees (Деревья поведения). Состояния: `Idle`, `Chase`, `Attack`, `Flee_at_low_HP`, `Call_For_Help`.
*   **Probability Tables:** Таблицы вероятностей критов и промахов зависят от разницы уровней и показателя "Удача" (Luck) + "Ловкость" (AGI).
*   **AI Combat Tuning:** Adaptive Difficulty может "налету" повысить HP босса на 20%, если группа сносит его быстрее ожидаемого Time-To-Kill, объяснив это лорно "Босс впитал темную энергию комнаты".

### 3.4. Scale-up логика
*   **Лимиты:** Зона в открытом мире (Hub) вмещает до 100 игроков (Soft cap).
*   **Инстансинг:** При превышении лимита (101-й игрок) сервер бесшовно создает "Зеркало" `Hub_Region_02`.
*   **Балансировка:** HAProxy / Nginx Ingress Controller распределяет TCP/WebSocket коннекты по микросервисам Room-серверов. (Используется архитектура Kubernetes, Redis Pub/Sub для межсерверного взаимодействия).

## 4. Интеграция с блокчейном Stellar

Stellar выбран как основная сеть (Protocol 25 / Soroban).

### 4.1. NFT-предметы (Loot as Smart Contracts)
*   **Механизм генерации:** Когда выпадает Редкий/Эпический/Легендарный предмет, Backend + ИИ формируют структуру параметров (Урон + Перки) + Текстовую Легенду ИИ.
*   **Обертка в формат Soroban:** Генерация JSON-метаданных `{"name": "Меч Тысячи Историй", "image": "ipfs://CID", "attributes": [...], "ai_history": "Этот клинок выковал..."}`. Загрузка в IPFS (через Pinata API).
*   **Чеканка (Minting):** Игрок через интерфейс нажимает "Mint NFT" -> Freighter Wallet просит подписать транзакцию к смарт-контракту NFT Коллекции на Soroban. Метаданные (или URI) записываются в блокчейн. При создании NFT становится `Transferable`.
*   **Передача:** Стандартная P2P передача ассетов (или Soroban-маркетплейс контракт "Escrow").

### 4.2. On-chain история мира (Achievements & Chronicle)
Не каждое убийство пишет транзакцию (это дорого и медленно). В блокчейн пишутся только эпохальные достижения ("Первое Убийство Лича-Короля", "Выбор: Сжечь Город эльфов").
*   **Формат хранения:** Смарт-контракт `Chronicle_Contract`. Функция `record_event(hash_id, event_type_enum, participants_pubkeys, minimal_compressed_data)`.
*   **Структура событий:** `0x01` (Quest Completed), `0x02` (World Decision).
*   **Рентген-лог квестов:** Игроки в конце сезона (или данжа) получают On-chain Badge (Non-Transferable NFT - SBT, Soulbound Token), символизирующий их вклад в лор сервера.

### 4.3. ZK-механики (Protocol 25 / X-Ray)
Stellar X-Ray представил host functions для `BN254` и `Poseidon/Poseidon2`. Это открывает "Privacy-Preserving Game Mechanics".
*   **Сценарий (Скрытое прохождение квеста / Stealth Heist):** Игрок получает квест на кражу Артефакта из гильдии убийц. Игрок не хочет, чтобы в публичном on-chain логе (и в клиенте других игроков-шпионов) был виден его маршрут, какие замки он взломал и какие скиллы применил.
*   **Офчейн-процесс:** Игрок решает лабиринт (загадку) локально в клиенте. Браузер (используя SnarkJS compilations) генерирует zk-SNARK доказательство того, что: "Я, зная начальный State A комнаты и правила коллизий, произвел N валидных ходов и пришел к State B (Выход из комнаты с сундуком)".
*   **Ончейн-валидация:** В смарт-контракт (Host Function BN254) отправляется Proof + Public Inputs (Hash начальной и конечной точки). Контракт валидирует Proof за долю комиссий. Никто в сети Stellar не узнает, КАКИМ ИМЕННО путем прошел игрок, но факт честного прохождения (ZK Proofs on Stellar X-Ray) записан навсегда. Награда выдается смарт-контрактом. Воровство лора без палева.

### 4.4. Голосование
В критические моменты (казнить NPC или помиловать), появляется окно голосования.
*   **Механика:** Транзакция вызова (Call) контракта `VoteSession`. 
*   **Веса голосов:** 1 Аккаунт = 1 Базовый Голос + Множитель. Если у игрока в кошельке (проверяется контрактом) есть Soulbound Token "Лидер Фракции", вес голоса = 3.
*   **Логика ИИ:** После истечения таймера (30 сек) Backend читает статус контракта. ИИ принимает победивший "Outcome" и мгновенно перестраивает следующую сцену под это решение.

## 5. Предметная система и лут

### 5.1. Классификация
1.  **Оружие (Weapon):** `[Sword, Bow, Staff, Dagger]`. Влияет на тип анимации, радиус атаки (RNG) и атрибут скейла урона.
2.  **Артефакты (Artifacts):** Броня (Шлемы, Доспехи), Щиты, меняющие защитные параметры.
3.  **Талисманы (Talismans/Trinkets):** Бижутерия. Дают уникальные Ауры ("10% Вампиризма в радиусе 5м", "Пассивный иммунитет к оглушению"). Ограничение: 2 слота.
4.  **Материалы (Materials):** Медь, кожа, руны. (Non-NFT, хранятся на сервере игры для дешевизны экономики).
5.  **Сюжетные (Quest/Lore Items):** Ключи, Древние Свитки. Не подлежат торговле. Привязываются к Party.

### 5.2. Алгоритм генерации
Структура (Backend RNG с весами):
*   **Редкость (Quality Level):** Common (60%), Uncommon (25%), Rare (10%), Epic (4%), Legendary/Mythic (1%).
*   **Характеристики:** `Base_Stat * Rarity_Multiplier`.
*   **Перки:** (Генерируются из пула в зависимости от редкости). Для Epic+ может добавиться "Скрытая способность" ("10% шанс призвать Метеорит при ударе").

### 5.3. Легенда предмета (AI-Lore Minting)
Уникализация предмета при дропе рангом Rare и выше.
*   **Структура легенды:** `[Кто создал/Кому принадлежало] + [Ключевое событие прошлого] + [Упоминание текущих игроков, нашедших его]`.
*   **Стили (Задаются ИИ в зависимости от биома):** "Мифический" (Эльфийские леса), "Бытовой/Грязный" (Канализация, бандиты), "Исторический" (Древние руины).
*   **Влияние игрока:** Если босс был убит только заклинаниями огня, ИИ-Мастер вплетает это: *"Раскаленный Клинок Очищения, рожденный в пламени, которым группа {Имя Танка} испепелила стража подземелья."*

## 6. Система прогрессии игроков

### 6.1. Характеристики (Attributes)
Основа механики: D20-подобные проверки на сервере.
*   **Сила (STR):** Урон ближнего боя, переносимый вес (Кол-во ячеек лута).
*   **Ловкость (AGI):** Скорость перемещения (WS velocity), уклонение, шанс крита, урон стрелкового оружия.
*   **Интеллект (INT):** Магический урон, реген маны. *Для ИИ:* дает подсказки в загадках (доп квесты).
*   **Харизма (CHA):** Влияет на цены у NPC, *Для ИИ:* открывает мирные ветки в диалогах, успех убеждения.
*   **Удача (LCK):** Вероятность Rare+ дропа, вероятность спастись при смертельном уроне на 1 HP.

### 6.2. Навыки и билд-система (Classless System)
*   **Древо умений:** Паутинообразное древо (похожее на Path of Exile, но меньше масштабом). Открывается за "Сферы Знаний" (Очки опыта).
*   **Активные способности:** Игрок может поместить в Action Bar только 4 активных скилла (Q, W, E, R). Выбор определяет "Билд" перед входом в зону (Deck building механика).
*   **Механика синергий (Synergies):** Если Скилл 1 - "Облить Водой" (Вешает дебафф 'Wet'), а Игрок 2 использует "Удар Молнии" в эту же цель = Урон x2 (Chain Lightning AoE/Combo effect). Поощряет Party-взаимодействие.

### 6.3. Адаптивный баланс
ИИ-Мастер регулярно опрашивает базу данных "World State". Если 80% активных онлайновых игроков имеют средний уровень (Gear Score / DPS) выше нормы биома — ИИ-Мастер автоматически поднимает глобальный уровень сложности (Global Threat Level +1). Это значит, что у всех новых мобов статы увеличиваются на X%, а шанс дроба ценных наград — на Y%. Баланс всегда "бросает вызов" комьюнити.

## 7. Пользовательский интерфейс (UI)

*Стиль: Минималистичный интерфейс (HUD), "Dark Fantasy / Terminal Aesthetic" — не отвлекает от карты.*

### 7.1. Экран игры (Canvas/DOM)
*   **Карта (Центр 60% экрана):** Пиксель-арт (Tilemap) отображение персонажей, местности, FX-эффекты магии.
*   **Боковая панель (Лево 20%):** Текстовый ЛОГ ИГРЫ (AI Dungeon Master chat), где детально, с эффектом печатной машинки, ИИ описывает происходящее: `"Воздух здесь пахнет озоном и древней смертью..."`
*   **Всплывающие события (Центр Modal Overlay):** Модальное окно квеста с описанием от ИИ и кнопками выбора (А, Б, В) с прогресс-баром таймера.

### 7.2. Панель действий
*   **Низ экрана (Bottom Bar):** 4 слота активных умений с таймерами кулдауна (ММОРПГ стиль). Рядом полоски HP (красная), MP (синяя) и Stamina (желтая).
*   **Чат (Рядом с логом):** Диалоги Игроков + Поле для инпута "Ответить NPC" или "Сделать действие" (Передается в Narrative Director/LLM).

### 7.3. NFT-инвентарь
*   Вкладка "Рюкзак". Предметы-NFT имеют золотое/фиолетовое свечение каемки (CSS `box-shadow`).
*   При клике — открывается `Lore Panel`: Описание от ИИ, On-chain History предмета.
*   **Кнопка [Mint NFT (Stellar)]**: Если предмет найден, но еще не заминчен — игрок платит комиссию в сети Stellar (Freighter), подписывая транзакцию, после чего предмет уходит из серверной БД и переходит в статус "On-Chain Ownership" в кошелек игрока.

## 8. Система квестов и сюжетных веток

### 8.1. Формальные типы квестов
Все квесты генерируются ИИ, но ложатся на формальные "движковые" протоколы, чтобы игра знала, как их валидировать:
1.  **Исследование:** Триггер `Explore_Area(Room_Nodes_Count > 5)`.
2.  **Уничтожение:** Триггер `Kill_Entity(Boss_ID)`.
3.  **Дипломатия** (Уникальная фишка): Триггер `Convince_NPC(Target_ID, "Surrender")`. Игрок пишет текст, LLM оценивает убедительность (с учетом броска CHA) и меняет стейт NPC на 'Surrendered'.
4.  **Спасение/Эскорт:** Триггер `Move_Entity_To(NPC_ID, XY_Target)`.
5.  **PvE-рейды:** Сложные многофазовые задачи с 2+ типами целей (Активировать Руны -> Убить Босса).

### 8.2. Структура квеста
Внутренний JSON-объект, который возвращает ИИ:
*   **Цель (Objective):** Формализованное условие движка.
*   **Контекст (Story):** Литературное описание для игрока.
*   **Препятствия:** Ограничение видимости, спавн особых юнитов, ловушки.
*   **Сюжетные развилки (Branches):** Если Провал -> Ветка B. Если Успех -> Ветка А.
*   **Привязка к игрокам:** ИИ-Мастер интегрирует имена игроков в квест: "Демон, который убил твоего согильдийца на прошлой неделе, теперь здесь".

### 8.3. Механика коллективных решений
*   **Консенсус:** Если группа из 5 человек, решение принимается по большинству голосов.
*   **Скрытые голоса (Secret Sabotage):** Игрок-ассасин может иметь скрытый класс-перк (Saboteur). В диалогах с NPC он может послать в WS тайный сигнал "Украсть", пока группа голосует за "Купить". Если кубик удачи игрока (LCK) успешен — предмет украден, группа ничего не знает, а ИИ-Мастер добавляет последствия: "Пока вы торговались, кто-то из вас обчистил карманы торговца...".

## 9. Экономическая система

### 9.1. Валюта
*   **Silver (Off-chain):** Мелкая монета, дропающаяся отовсюду. Хранится на бэкенде. Для расходников.
*   **Токен (Stellar Native Token / DND_TKN):** Интеграция с блокчейном. Покупается за XLM / USDC на децентрализованных биржах Stellar (SDEX). Нужна для голосований (Governance), чеканки NFT, покупки слотов инвентаря.
*   **NFT (Asset):** Главный высоколиквидный инструмент экономики. Трейдится между игроками.

### 9.2. Экономика редкости (Anti-Inflation)
*   **Динамическая вероятность дропа:** ИИ имеет доступ к `Contract Data` (Сколько мечей 'Legendary Tier Type S' уже выдано). Если выдано больше 100 — шанс дропа падает по экспоненте асимптотически к нулю. Экономика никогда не наполняется топ-предметами.
*   **Сезонность (Seasons):** Раз в 3 месяца "Катаклизм" (Вайп мира). Старые NFT остаются в кошельках как "Коллекционные предметы с Лором Прошлого Века", в новом сезоне они превращаются в Cosmetic / Skin предметы, заставляя игроков добывать лут заново. Спрос поддерживается всегда.

### 9.3. Межпользовательский Аукцион
*   **P2P-торговля:** Интерфейс в игре (Вкладка Аукцион).
*   **Ончейн механика:** Использование Soroban Smart-Contracts для создания предложений на продажу (Maker/Taker модель DEX-а) за XLM или Native Token. 
*   Каждый NFT (Лот) отображает свою AI-легенду, что повышает ценность (предмет с интересным лором / лором популярного в комьюнити убийства может стоить дороже обычного аналога).

## 10. Безопасность и анти-чит

### 10.1. Проверка событий на сервере (Server Authority)
*   Браузер отправляет только *Inents* ("Хочу идти туда", "Нажимаю Атаку"). Сервер (Node.js/Rust) сам пересчитывает тайлы, проверяет кулдауны, вычитает ману и отправляет клиентам результат.
*   Damage Manipulation, Speedhack, No-clip технически невозможны (сервер откинет игрока обратно — Rubberbanding, если координаты изменяются быстрее лимита).

### 10.2. ZK-доказательства честности (Protocol 25)
Вместо отправки всех промежуточных шагов на блокчейн для аудита, игра может использовать ZK.
*   Для турнирных, высокоуровневых квестов, Игрок локально формирует ZK-SNARK (весом 300-500 байт), доказывая, что он: 1) Имел достаточное количество HP перед боем; 2) Не использовал эксплойты в расчетах; 3) Нанес ровно Х урона. Сервер верифицирует легкий Proof, исключая любые манипуляции клиентом доверяемых офлайн-секьюрити блоков.

### 10.3. Защита от ботов (AI + Captcha)
*   **Анализатор поведения:** Если Player Analyzer видит, что координаты кликов и тайминги идеальны в течение 2 часов (Типичный Bot-loop для фарма золота).
*   **Реакция:** ИИ-Мастер *не* банит сразу, он генерирует на пути бота "NPC Хранителя" (или "Пространственную Аномалию") и инициирует сложный, контекстный диалог. Боты-макросы не смогут пройти тест Тьюринга от LLM. Если бот проваливает диалог (или игнорирует его n-раз), он отправляется в Тюремную Зону (Jail Bioome), выбраться из которой можно только решив уникальную AI-загадку.

## Финальное системное резюме (Development Prompt)

**Архитектурная схема (Высокий уровень):**
1.  `[Браузерный Клиент (React + WebGL(Phaser) + Freighter Wallet)]` <-> `[WSS/HTTPS (Nginx)]`
2.  `[Nginx]` <-> `[Game Server Pool (Node.js/Rust) -> World State (Redis + Postgre)]`
3.  `[Game Server]` <-> `[AI Orchestrator (Python FastAPI -> RAG VectorDB + LLM API)]` -> Генерация контента.
4.  `[Game Server/Client]` <-> `[Stellar Horizon / RPC Node]` -> Вызов Soroban-контрактов для NFT и ZK Proofs (Protocol 25 X-Ray).

Этот документ служит исчерпывающим ТЗ для старта разбивки на пользовательские истории (User Stories) и спринты как для Blockchain-разработчиков (Soroban/Rust), так и для Game-Backend (Node.js), Frontend (React/Phaser) и AI Integration инженеров. Модель масштабируемая, безопасная и в полной мере реализует концепцию адаптивного AI-Мастера Игры в сети Stellar.
